# 部署到服务器（一次部署，所有人开网址即用，永不掉线，无需终端）

> 解决：本地跑要开终端、关窗口/重启就掉线、发给别人每人都要装。部署到公网服务器后 —— 大家只打开一个**网址**，没有终端、不掉线、零安装。

---

## 先定一个事：用谁的 key？

- **方案甲（你统一出 key）**：服务器上配你的 `KUAIZI_API_KEY`/`IMAGE_API_KEY`。用户零配置、打开就用，但**丽帧出片费用由你承担**（适合你自己的产品/对外 SaaS，配合登录/限额）。
- **方案乙（各用户填自己的 key）**：页面让用户填自己的丽帧 key、随请求带上，服务器用它去调。**费用各付各的**。需要我改一版"页面填 key→后端透传"的逻辑（约半小时）。

下面部署步骤两种方案通用，区别只在"key 放服务器还是用户填"。

---

## 方式 A · 最省事：连 GitHub 一键部署（Railway / Render / Fly.io 等）

你的代码已在 GitHub（VIDEO-FACTORY）。以 Railway 为例：

0. **先把最新代码推到 GitHub**：双击项目里的 `push-to-github.bat`（把本次新增的 作品页/双语/锁一致性/Dockerfile.live 等一并推上去；你的 `.env` 与 `hq/` 已被 .gitignore 排除，不会上传）。
1. railway.app 注册登录 → New Project → Deploy from GitHub repo → 选 `VIDEO-FACTORY`。
2. 部署方式选 **Dockerfile**，路径填 `Dockerfile.live`（带 ffmpeg）。
3. Variables（环境变量）——**你选的"每用户填自己 key"模式**：
   - **不要**设 `KUAIZI_API_KEY`：页面会自动提示每个用户填自己的丽帧 key，视频费用各付各的。
   - 可设 `IMAGE_API_KEY=sk-...`（你的网关 key，让"免描述写脚本 + 锁人物/产品三视图"对所有用户可用，成本低、由你出）；也可留空让用户连这个一起填。
   - 设 `WEB_PORT=8088`。
4. Settings → Networking → Generate Domain → 得到一个公网网址（形如 `xxx.up.railway.app`）。
5. 打开那个网址即用；发给任何人都能用。Render/Fly 流程类似（都支持 Dockerfile + 环境变量 + 自动 HTTPS）。

> 免费档磁盘是临时的：重部署后"我的作品"(works.json)和成片会清空。要长期保存，挂一个持久卷或把成片转存对象存储（OSS/S3）——需要我加，告诉我即可。

## 方式 B · 自己的 VPS（阿里云/腾讯云轻量，约几十元/月，最可控）

服务器装好 Docker 后：
```bash
# 把项目传上去（git clone 你的仓库，或 scp 整个文件夹）
cd universal-video-generator
docker build -f Dockerfile.live -t video-factory .
docker run -d --name vf --restart always -p 80:8088 \
  -e KUAIZI_API_KEY=kz-你的 \
  -e IMAGE_API_KEY=sk-你的 \
  -v /opt/vf-out:/app/.uvg-out \
  video-factory
# 浏览器开 http://你的服务器IP  即用；绑域名 + 上 HTTPS 更正式
```
`--restart always` = 开机/崩溃自动拉起，永不掉线。`-v` 把成片/作品挂到宿主机，重建容器不丢。

**不想用 Docker** 也行（VPS 装 Node 20 + ffmpeg）：
```bash
npm i -g pm2
pm2 start live-server.js --name vf      # pm2 守护，崩溃自动重启
pm2 startup && pm2 save                  # 设为开机自启 → 永远在跑
```

---

## 环境变量
| 变量 | 必填 | 说明 |
|---|---|---|
| `KUAIZI_API_KEY` | 是 | 丽帧视频 key（kz-）。方案乙改为用户填则可不设 |
| `IMAGE_API_KEY` | 建议 | 网关 key（sk-）：免描述写脚本 + 三视图锁人物/产品 |
| `WEB_PORT` | 否 | 默认 8088；Docker 里映射到 80/443 |
| `LLM_MODEL` | 否 | 默认 gpt-5.5 |

## 安全
- key 只放服务器环境变量，**绝不写进网页或仓库**（仓库已 .gitignore 掉 .env）。
- 上 HTTPS（Railway/Render 自带；VPS 用 Caddy/Nginx 一键证书）。
- 对外开放建议加：登录/限额/每日上限（项目 `server.ts` 有多租户+配额可借鉴），防止被刷爆丽帧费用。

## 持久化（可选但推荐）
作品记录 `.uvg-out/works.json` 与成片 mp4 默认存本地磁盘。云平台要挂持久卷（方式B的 `-v`），或改存对象存储（OSS/S3）——需要我接。
