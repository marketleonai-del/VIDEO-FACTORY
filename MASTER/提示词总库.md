# 提示词总库（复制即用，{} 为变量）

> 全部提示词集中于此，按全流程顺序排列。来源：`live-server.js`、`ugc-creative-amplifier/SKILL.md` 与 `references/*`。用法见《视频工厂-总纲》§3。

## 1. 产品分析 / 卖点提取  【提示词:产品分析】
> 调用网关 LLM（gpt-5.5），可附产品图（视觉）。提取"产品核心"。

```
system: 你是资深带货操盘手。只输出 JSON，不要解释。
user: 分析这个产品并输出 JSON：
{"品类":"","核心痛点":["",""],"核心卖点":["","",""],"目标人群":"","价格带":"","认知误区":"","竞品通常打的角度":""}
产品：{产品名 或 见图}
```
（看图：user 用数组 `[{type:"text",text:上文},{type:"image_url",image_url:{url:"data:image/...base64"}}]`）

## 2. 带货脚本（钩子→卖点→效果→促单）  【提示词:带货脚本】

```
system: 你是短视频带货编剧。输出一条 {时长}s 竖屏带货口播脚本，结构：前3秒钩子 → 核心卖点(可视化) → 使用/效果 → 促单CTA。
口语、有画面感、无绝对化用语、无未证功效。只输出脚本正文。
user: 产品：{产品}；卖点：{卖点}；人群：{人群}；平台：{抖音/TikTok}。
```

## 3. 故事板 / 分镜（★先故事板再出片）  【提示词:故事板分镜】

> A. 引擎实际在用的"自动分镜"提示（`live-server.js` llmScenes，免描述也能跑）：
```
system: 你是短视频带货导演。只输出一个 JSON 数组（{N} 个中文字符串），每个元素是一个分镜的画面脚本（可直接喂给文生视频模型），递进体现：钩子→产品核心卖点→使用/效果→促单。不要解释，不要多余文字。
user: 产品/主题：{产品}。总时长约 {时长} 秒，分 {N} 段。每段一句，画面感强、适合竖屏带货。
```

> B. 完整故事板（每镜 9 字段，更专业，移交逐镜细化时用）：
```
system: 你是分镜导演。把脚本拆成 {N} 个分镜，输出 JSON 数组，每项：
{"镜号":1,"景别":"近景/中景/特写","运镜":"推/拉/摇/跟/固定","时长s":,"画面":"可直接喂文生视频的画面描述","台词":"该镜口播","素材":"真素材/AI","锚定":"@image1产品 等","转场":"硬切/叠化"}
分段规则：单镜≤15s；总时长{T}→镜数{N}并均摊时长。只输出 JSON。
user: 脚本：{脚本}；产品锚定：@image1={产品}。
```

## 4. 角度卡（6 大角度族）  【提示词:角度卡】  来源 hook-angle.md
> 6 族：A痛点放大 / B反常识 / C身份认同 / D省钱算账 / E信任背书 / F情绪共鸣。

```
system: 你是广告策略师。给产品扫 6 大角度族(A痛点放大/B反常识/C身份认同/D省钱算账/E信任背书/F情绪共鸣)，
产出 {N} 个"真正不同"的角度。每个输出 JSON：
{"角度族":"B反常识","角度名":"","钩子前3秒原话":"","为什么不同":"","目标人群":"",
 "WinScore":{"新颖0-25":,"痛点0-20":,"钩子0-20":,"平台0-15":,"信任0-10":,"可视0-10":,"合计":},"合规风险":"低/中/高"}
要求：跨族覆盖；角度两两重合度≤15%；按 WinScore 降序。只输出 JSON 数组。
user: 产品：{产品}；竞品通常打：{竞品角度}；平台：{平台}。
```
WinScore = 角度新颖度(0-25)+痛点强度(0-20)+前3秒钩子(0-20)+平台契合(0-15)+信任密度(0-10)+卖点可视化(0-10) − 同质化 − 合规风险。（透明先验，非真实预测；有 CTR/完播/GMV 后回填校准。）

## 5. 钩子公式库（前 3 秒）  【提示词:钩子库】  来源 hook-angle.md / hook-library-douyin-tiktok.md
```
痛点共鸣式： "你有没有过______？"（痛点具体到场景）
反常识式：   "别再______了，______才是对的"
数字冲击式： "我测了______款，只有1款______"
损失厌恶式： "还在用______？你每月多花______"
身份钩子式： "______的人，柜子里都有这个"
悬念式：     "这东西到底______？我用了30天告诉你"
对比式：     "左边______，右边______（同一时间）"
自嘲反向式： "劝你别买，除非你______"
```
红线：禁止"大家好今天推荐一款…"无效开场；前 3 秒必须有可见钩子。完整 10 族~80 钩子 + AI 友好度 + 矩阵轮换规则见 `../ugc-creative-amplifier/references/hook-library-douyin-tiktok.md`。

## 6. 文生/图生视频 画面提示 + 续帧后缀  【提示词:画面】
> 喂给丽帧 create 的 prompt（模型无关，Seedance2.0/可灵/Veo/Runway 通用）。
```
画面提示：{分镜.画面}，竖屏 9:16，电影感运镜({分镜.运镜})，明亮高级，真实质感
通用后缀：, 同一主体一致, 连续运镜, 同风格, 无文字水印, no caption
续帧（多段一致）：第1段 create 传 return_last_frame:true；后续段 images:[{url:上段尾帧, role:"first_frame"}] + "衔接上一镜头, 主体与场景保持一致"
```

## 7. 配音 / 情绪  【提示词:配音】
> 本地 TTS（零云成本），全片单一声纹 @voice1，AI 画面静音、统一音轨。
```
配音文本 = 各分镜台词顺次拼成一段连续口播；情绪 ∈ {中性,开心,严肃,温柔,激动}
要求：口语自然、有呼吸感、非"AI 腔"；语速适配平台({抖音偏快})。
```

## 8. 矩阵差异化（跨账号去重）  【提示词:矩阵差异化】  来源 matrix-content.md
```
system: 把 1 个赢家概念放大成 {N} 条账号安全变体。每条输出 JSON：
{"变体号":,"账号人设":"","平台":"","钩子变体":"(从钩子库换族)","差异维度":"改了什么(人设/开场镜/BGM/节奏)","跨账号重合度":"≤20%","发布时段":"","CTA":""}
要求：复用精品母版"身体"只换钩子；跨账号重合≤20%；开场镜/BGM/字幕各异(防限流)。只输出 JSON 数组。
user: 赢家：{母版概念}；平台：{平台}；账号数：{N}。
```

## 9. 质检 120 门 + 合规卡  【提示词:质检】  来源 SKILL.md
```
【放大引擎自检 · 120分制，<100 自动重修(≤3轮)】
内容(30): 角度真不同 / 钩子前3秒成立 / 卖点可视化
真实(25): 信任点用真素材 / 无恐怖谷 / 非完美细节保留
多样(20): 段内≤15% / 跨账号≤20% / 维度覆盖
合规(20): AI标识 / 分区清单 / 零禁用词 / 功效合规
闭环(15): 每条带WinScore / 预留 actual{CTR,完播,GMV} 回填字段
技术(10): 模型无关提示词 / @image锚定 / 一致性约束
```
```
【合规卡】（每次输出强制附带）
□ AI生成内容标识：已附（中国《标识办法》2025-09-01 起强制）
□ 分区：中国(显著标识+元数据)/欧盟(AI法透明度)/Meta·TikTok(AI人物披露)
□ 话术：无绝对化用语 / 无未证功效 / 非医疗不提疗效 / 效果标"因人而异"
□ 真实性：禁假证言/假原价/假倒计时
```
```
【人感护栏】（混剪/有口播时）：口播自然度 / 保留非完美真实细节 / 节奏呼吸 / 是否过度AI化
```
