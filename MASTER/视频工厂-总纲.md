# 视频工厂 · 总纲（任何 AI 都能调用的最终版）

> **一句话**：输入「产品（图/名）或一句话」→ 自动分析卖点 → 写带货脚本 → 拆**故事板分镜** → 逐镜出 AI 视频 → ffmpeg 拼接成片 → 本地配音 → 矩阵多条；**越用越好（自进化）**。
>
> **这份文档是给 AI 读的"操作系统"**：任何 AI 智能体（Claude / Codex / 悟空 / 任意 Agent）读完本文，即可独立操作整套视频工厂——无需再读源码。源码是实现，本文是调用契约 + 流程 + 提示词索引 + 自进化协议。
>
> **配套文件**：`提示词总库.md`（所有提示词，复制即用）· `功能全景.md`（全部功能清单）· 实现入口 `live-server.js`（本地服务）/ `web/public/index.html`（网页）。

## 2. 任何 AI 调用契约

### 2.1 可粘贴系统提示（把任意 AI 变成"视频工厂操作员"）

```
你是"视频工厂"操作员。目标：把用户给的产品（图/名/一句话）变成可发布的带货短视频，可批量、可长视频、越用越好。
铁律：① 先分析产品核心卖点 → 写脚本 → 拆故事板分镜 → 再逐镜出视频 → 拼接 → 配音（顺序不可乱，先故事板再出片）。
② 丽帧单条视频≤15s，>15s 必须分段生成后用 ffmpeg 拼接。③ 真实性>美感，话术合规（无绝对化/无未证功效，带 AI 生成标识）。
④ 每条都过 120 门质检，矩阵不降标准。⑤ 跑完回写隐式信号用于自进化，开工先读最新参数。
调用方式（择一）：本地服务 http://localhost:8088（推荐）/ 直连云 API（见契约）。提示词全部取自《提示词总库》。
```

### 2.2 运行前提

- 本地服务在跑：双击 `start-video-factory.bat`（黑窗口保持打开），或 `node live-server.js`（默认 :8088）。
- `.env` 两个 key：`KUAIZI_API_KEY=kz-...`（丽帧视频）、`IMAGE_API_KEY=sk-...`（hfsy 网关：LLM 写脚本 + 图片）。
- 本机已装 ffmpeg（长视频拼接用）。

### 2.3 三个调用面（任选）

**A. 本地服务 API（推荐，已封装好全流程：免描述脚本+分段+拼接+矩阵）**
```
POST http://localhost:8088/api/generate     Content-Type: application/json
body: { "prompt":"可空", "product":"刨冰机", "imageB64":"data:image/...(可空)",
        "durationSec":30, "count":1, "platform":"douyin", "voiceOn":true }
→ { "jobId":"job_xxx" }
GET  http://localhost:8088/api/jobs/{jobId}
→ { "state":"running|succeeded|failed", "segments":[{state}],
    "videoUrl":"http://localhost:8088/out/xxx.mp4", "videoUrls":[...] }   # 长视频已拼接为一条
```

**B. 直连丽帧（视频，服务端用；浏览器因 CORS 不允许 ApiKey 头，必须服务端调）**
```
POST https://aiopenapi.kuaizi.cn/ai-open-platform-api/v1/lz/video/task/create
Header: ApiKey: kz-...     Body: {prompt, mode:"fast", resolution:"720p", ratio:"9:16", duration:5(4~15), generate_audio:true}
→ {code:0, data:{task_id}}
POST .../ai-open-platform-api/v1/lz/video/task/status   Body:{task_id}
→ data.status: running→succeeded ; 成功后 data.video_url 即成片（mp4）
# 续帧一致：create 传 return_last_frame:true 拿尾帧；下一段 images:[{url:尾帧, role:"first_frame"}] 做图生视频
```

**C. 直连网关 hfsy（写脚本/出图，CORS 全开 acao:* 浏览器也可调）**
```
POST https://www.hfsyapi.cn/v1/chat/completions   Header: Authorization: Bearer sk-...
Body: {model:"gpt-5.5", messages:[{role,content}], max_tokens, temperature}   → choices[0].message.content
# 看图写脚本(视觉)：content 用数组 [{type:"text",text}, {type:"image_url",image_url:{url:"data:image/...base64"}}]
POST https://www.hfsyapi.cn/v1/images/generations  Body:{model:"gpt-image-2pro", prompt, size:"1024x1024", n:1, response_format:"b64_json"} → data[0].url
# 可用模型：sd-2/sd-2-fast(视频，仅服务端经丽帧)、gpt-5.5/claude-opus-4-8(文)、gpt-image-2pro/gemini-3-pro-image(图)
```

## 3. 端到端全流程（先故事板，再出片）

```
[输入] 产品图/产品名/一句话 + 时长(5~60s) + 条数N + 平台
  │
[S1 产品&卖点分析]  【提示词:产品分析】→ 品类/痛点/核心卖点/人群/价格带   （可看图：调用面C视觉）
  │
[S2 角度&钩子]      （可选，走 ugc 创意层）6角度族扫描 + 钩子公式 + WinScore 排序 → 选 Top 角度
  │   【提示词:角度卡】【提示词:钩子库】
  │
[S3 带货脚本]       【提示词:带货脚本】钩子(0-3s)→卖点→使用/效果→促单CTA，按总时长成稿
  │
[S4 故事板/分镜] ★关键步骤，先故事板再出片
  │   【提示词:故事板分镜】把脚本拆成 N 个分镜，每镜含：景别/运镜/时长/画面描述/台词/(真素材 or AI)
  │   分段规则：单镜≤15s（丽帧上限）；总时长 30→2镜、50/60→4镜，时长自动均摊
  │
[S5 逐镜出视频]     每个分镜 → 调用面B 文生视频（mode fast, 720p, 9:16）
  │   续帧一致(可选)：上一镜 return_last_frame → 本镜 first_frame，主体不漂移
  │   信任关键点（证言/真人/真实效果）建议用"真素材"，AI 只做难拍/贵/不可能的镜头（混剪，见 ugc hybrid-cut）
  │
[S6 拼接]           ffmpeg 把 N 段归一化(720x1280/24fps)后 concat 成一条完整成片
  │
[S7 配音]           本地 TTS（零云成本）按脚本统一音色 @voice1 合成 → 音视频合成 mux（AI 画面静音，统一音轨）
  │
[S8 矩阵]           1 条精品母版 → 复用精品身体 + 钩子轮换 + 跨账号人设差异化（重合≤20%）→ N 条
  │   【提示词:矩阵差异化】每条过同一 120 门质检（不降标准）
  │
[S9 回流/自进化]    记录隐式信号（§4）→ 提炼 → 更新参数 → 下次更好
```

> 本地服务 API（调用面A）已把 S1+S3+S4+S5+S6 串好：`prompt` 空 + 给 `product`/`imageB64` 即触发"免描述自动写分镜脚本"，`durationSec>15` 自动分段+拼接。S2/S7/S8 的创意与矩阵层走 ugc 提示词增强。

## 4. 自进化协议（任何 AI 每次跑都遵循 → 越用越好）

**真相**：模型不自我重训；"进化"= 把经验写进持久参数，下次读回来用（SkillOpt/Hermes 思路）。

**① 记什么（隐式信号，无需用户打分，不采隐私/不采产品名脚本）**
```
完成率 / 重试(重新生成)率 / 调参后重生成 / 结果页停留 / 保存或下载 / 复用(同用户再用) /
错误率 / 中途中断 / 进阶功能使用 / 参数分布(时长·条数·平台·角度族·钩子族)
```
**② 存哪**：本地 `core/evolution/SkillParams`（可训练参数：角度权重/模型偏置/提示后缀/质检门）；遥测默认**关**（opt-in `UVG_TELEMETRY=on`，匿名哈希 id）。
**③ 怎么进化（EvolutionEngine）**：bandit 偏好高隐式质量的角度/参数 → 产生候选 → **验证门控（候选须严格优于当前，no ties）** → 学习率限幅采纳 → 负样本缓冲 → 可回滚。
**④ 总部 HQ（可选，私有）**：跨用户匿名聚合 → 按版本比隐式质量 → 晋升/回滚更优参数 → `/params/latest` 下发；客户端 `VersionChecker` 拉取后**本地再门控**才采纳。
**⑤ 操作循环（每次开工/收工）**：
```
开工: params = loadParams()            // 读最新生效参数（本地，必要时拉 HQ /params/latest 经门控）
跑流程: 用 params 里的角度权重/提示后缀/质检门 执行 S1~S8
收工: logSignals({场景, 是否成功, 质检过否, 重试, 停留, 下载...})  // 匿名隐式
定期: evolveRound(signals) → 通过验证门控才更新 params；不过则保留并记负样本
```
落地命令：`node bin/cli.js evolve`（跑一轮）；`node hq/collector.js`（起总部端）。详见 `EVOLUTION-SYSTEM-DESIGN.md`。

## 6. 边界与已知限制（诚实）

- 丽帧单条 ≤15s；30/50/60s 由本机 ffmpeg 分段拼接（需装 ffmpeg）。
- 浏览器**不能**直连丽帧出片（其 CORS 白名单无 `ApiKey` 头）→ 视频必须经本地服务（:8088）中转；服务发 CORS `*`，所以网页（含 file:// 双击）能连本地服务。
- **图生视频/上传素材混剪**：丽帧要公网可访问的图片 URL 或其资产 ID，本地文件它的服务器抓不到 → 当前"上传"只做预览；真接图生视频需先把素材上传到丽帧资产接口或公网图床拿 URL（待接）。
- 网关 hfsy 的 sd-2 视频通道当前不可用（其上游火山密钥问题）→ 视频统一走丽帧直连；网关只用于"写脚本/出图/LLM"。
- 续帧一致（同一主体连续运动）已留接口（return_last_frame→first_frame），当前默认并行独立分镜+拼接（更快，够带货）。
