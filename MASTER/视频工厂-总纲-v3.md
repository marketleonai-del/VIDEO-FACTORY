## 0. 系统概述

VIDEO-FACTORY 是一个面向电商带货短视频的 **AI 视频生产工厂**。它接收一个产品（图片/名称/一句话），自动完成从卖点分析到成片输出的完整链路，支持矩阵批量生产，并通过 **300-Agent 蜂群架构** 实现规模化协作，通过 **SkillOpt 自进化机制** 实现越用越好。

### 核心能力矩阵

| 能力 | 说明 | 状态 |
|------|------|------|
| 产品洞察 | 看图识品 + 结构化卖点分析 | ✅ 已上线 |
| 角度钩子 | 6大角度族 + 10大钩子族(80条) | ✅ 已上线 |
| 故事板分镜 | 脚本 → 分镜 → 逐镜出视频 | ✅ 已上线 |
| AI 视频生成 | 丽帧/可灵/Seedance 统一接口 | ✅ 已上线 |
| 混剪 | 真素材 + AI 素材智能混剪 | ✅ 已上线 |
| 矩阵 | 1条精品 → N条差异化变体 | ✅ 已上线 |
| 声纹锁 | @voice1 全片音色统一 | ✅ 已上线 |
| 质检 | 120门质量检测，<100重修 | ✅ 已上线 |
| 自进化 | SkillOpt 参数自动优化 | ✅ 已上线 |
| 蜂群 | 300-Agent 分布式协作 | ✅ 已上线 |
| 长视频 | 多段续帧拼接(5-60s) | ✅ 已上线 |
| 本地TTS | 零云成本配音 | ✅ 已上线 |

### 技术特征

- **纯 Node.js**，零运行时依赖（仅 Node 内置 + ffmpeg 系统二进制）
- **模型无关**：新增视频/TTS 模型只需写一个 Adapter
- **隐私优先**：上报默认关闭（opt-in），匿名哈希 ID，白名单字段过滤
- **成本可控**：AI 只做 0-3s 钩子 + 真素材混剪，省 90% 成本
- **质量不降**：精品母版法，矩阵逐条过同一 120 门质检

### 1.2 角色分配（总计 300 Agent）

| 角色 | 数量 | 职责 | 对应模块 |
|------|------|------|----------|
| **generator** | 200 | 视频生成 Agent，执行带货视频生成、素材下载、ffmpeg 拼接 | core/video/*, core/audio/* |
| **evolver** | 50 | 进化 Agent，运行本地进化算法、上报隐式信号、调优参数 | core/evolution/* |
| **qa** | 30 | 质检 Agent，执行 120 门质量检测、合规检查、人感检测 | core/quality/* |
| **creative** | 20 | 创意 Agent，优化提示词、拍摄角度、钩子生成、分镜设计 | core/creative/* |

### 1.3 Agent 生命周期

```
注册(registerAgent) → 心跳(heartbeat) → 任务接收(assignTask) → 执行 → 结果上报(collectResult) → 心跳 → 注销(unregisterAgent)
```

- **注册**：Agent 启动时向 HQ 注册，分配角色和能力标签
- **心跳**：每 15-30 秒发送心跳，HQ 更新状态，检测版本是否落后
- **任务分配**：HQ 按负载均衡策略分配任务，优先选择 idle Agent
- **结果收集**：Agent 完成任务后上报结果，HQ 提取隐式信号
- **健康检查**：HQ 每 15 秒检查心跳超时，自动重分配离线 Agent 的任务

### 1.4 批量任务分解

以 200 条视频生成为例：
- 将 200 条视频拆分为 200 个子任务
- 每个子任务分配独特的 **角度 × 钩子 × 人设** 组合（确保多样性）
- 12 个角度（开箱测评/痛点解决/场景展示/对比实验/价格解析/使用教程/效果见证/成分科普/穿搭搭配/送礼推荐/限时福利/真实反应）
- 12 种钩子（震惊型/悬念型/数字型/反转型/共鸣型/利益型/情感型/权威型/对比型/挑战型/故事型/紧急型）
- 12 种人设（专业测评师/邻家姐妹/理性消费者/潮流达人/省钱高手/品质生活家/新手小白/资深用户/宝妈推荐/学生党/职场精英/健康生活家）

### 1.5 蜂群健康度

HQ 实时计算蜂群健康度（0-100）：
- Agent 在线率 30%
- 利用率 25%
- 队列健康 25%
- 角色平衡 20%

## 2. SkillOpt 自进化机制

### 2.1 核心思想

模型不自我重训；"进化" = 把经验写进持久参数，下次读回来用。融合 **SkillOpt**（Microsoft，SKILL.md 即可训练参数 + 验证门控 + 文本学习率）、**Hermes**（Nous Research，用后即学）、**GenericAgent**（小种子长出能力）的核心思想。

### 2.2 两级进化架构

**第一级 · 本地快速个性化（快、私有、个性化）**
- 每次生成的隐式事件留在本机
- `EvolutionEngine.evolveRound()` 跑一轮：`propose()` → `validate()` 验证门控 → `applyLearningRate()` 限幅采纳
- 特点：响应快、贴合个人素材/品类、无需联网、不暴露数据

**第二级 · 总部全局最优（慢、稳、全局）**
- `Aggregator.ingest()` 汇集多客户端匿名信号
- `byVersion()` 按 `paramsVersion` 算 `computeQuality`
- `decide(minSamples=50, margin=0.02)` 判定 promote / rollback / hold
- 特点：样本大、抗噪、求群体最优

### 2.3 可训练参数（SkillParams）

```javascript
{
  angleWeights:   { A:1.0, B:1.0, C:1.0, D:1.0, E:1.0, F:1.0 },  // 6大角度族权重
  modelBias:      { kling:1.0, lizhen:1.0, seedance:1.0, veo:1.0 }, // 模型偏置
  hookTemplates:  { H1:1.0, H2:1.0, ..., H10:1.0 },              // 10大钩子族权重
  promptSuffix:   '手机实拍质感, no text',                        // 提示词后缀
  qcThreshold:    100,                                             // 质检门阈值
  version:        'v0-default'                                     // 参数版本
}
```

### 2.4 隐式信号（无需用户打分）

| 信号 | 字段 | 类型 | 含义 |
|------|------|------|------|
| 完成率 | `success` | bool | 事件类型是否成功 |
| 重试次数 | `retries` | int | 用户对同需求重做次数 |
| 重做标记 | `regenerated` | bool | 同需求重做 = 不满意 |
| 质检分 | `qcScore` | 数值 | 120 门质检分数 |
| 质检通过 | `qcPass` | bool | 是否通过质检 |
| 拼接成功 | `assembleOk` | bool | 长视频拼接是否成功 |
| 耗时 | `ms` | 数值 | 端到端耗时（代理停留） |
| 模型 | `model` | 类别 | 使用哪个模型 |
| 场景 | `scene` | 类别 | 角度族分类 |
| 角度 | `angle` | 类别 | 具体角度 |

### 2.5 进化流程（evolveRound）

```
propose(signals)          → 按隐式质量提出候选参数（ε-greedy）
  ↓
validate(candidate)       → 验证门控（样本/负样本/边界/基线/差异）
  ↓ 通过 → applyLearningRate() → 限幅采纳（lr=0.3）→ commit → 更新生效
  ↓ 拒绝 → 入负样本缓冲 → 保留当前参数 → 记录拒绝原因
```

### 2.6 验证门控（防跑偏核心）

候选必须**严格优于**当前（`improvement > 0`，no ties → 杜绝静默漂移）：
1. 样本量 ≥ minSamples（默认 10）
2. 负样本率 ≤ 阈值（默认 40%）
3. 参数在 [weightFloor, weightCeiling] 范围内
4. 基线质量 mean > 0.6
5. 候选与当前参数必须有实质差异

### 2.7 回滚机制

- 保留 `lastGood` 快照
- 坏进化一键 `rollback()` 回到上次通过的参数
- 总部侧：检测冠军显著回退（< 历史最佳 − 2·margin）时建议 rollback

### 2.8 隐私保护

- 默认关闭上报（`UVG_TELEMETRY=off`）
- `anonId` = `sha256(randomUUID()+salt).slice(0,16)`，与真实身份无映射
- `ALLOW` 白名单物理拦截未登记字段，杜绝误采 PII
- 传输只有隐式信号（成功/重试/重做/质检/拼接/耗时），不含脚本文本/素材/产品名/Key
- 开源可审计，一键关闭

```
[输入] 产品图/产品名/一句话 + 时长(5~60s) + 条数N + 平台

  │
  ▼

  → 输出：{rank, family, title, winScore, hooks[{hookId, template, filledExample}]}
  │
  ▼
[S3 带货脚本]       【提示词:带货脚本】
  → 钩子(0-3s) → 核心卖点 → 使用场景/效果 → 促单 CTA
  → 按总时长成稿，口语、有画面感、无绝对化用语
  │
  ▼
[S4 故事板/分镜]  ★ 关键步骤，先故事板再出片 【提示词:故事板分镜】
  → 把脚本拆成 N 个分镜，每镜含：镜号/景别/运镜/时长/画面描述/台词/素材(真 or AI)/锚定/转场
  → 分段规则：单镜≤15s（丽帧上限）；总时长 30→2镜、50/60→4镜，时长自动均摊
  → 产品三视图 + 人物三视图（图生图锚定一致性）
  │
  ▼
[S5 逐镜出视频]     【调用面 B】

  │
  ▼

  → 本地 TTS（零云成本）按脚本统一音色 @voice1 合成
  → AI 画面静音 → 整片口播用 @voice1 一次性合成/录制 → 贴到拼好画面上

[S8 矩阵]           【matrix-engine.js】【提示词:矩阵差异化】
  → 1 条精品母版 → 复用精品身体 + 钩子轮换 + 跨账号人设差异化（重合≤20%）→ N 条
```

## 6. 模块详细说明

### 6.1 钩子角度模块（hook-angle）

> 卖最稀缺的东西：**不同的「想法」**。市场原话——AI 不需要更多微调的广告，需要更多真正不同的角度、钩子与价值主张。

#### 6.1.1 工作流（6 步）

```
S1 产品&人群洞察 → S2 角度空间扫描(6族) → S3 钩子公式套用
→ S4 差异化筛选(重合度≤15%) → S5 WinScore排序 → S6 输出角度卡 + 移交建议
```

#### 6.1.2 S1 产品 & 人群洞察

提取 8 维信息：
- 品类 / 核心痛点(2-3) / 卖点类型 / 价格带 / 目标人群 / 转化目标
- **认知现状**：用户对该品类的既有认知/误区（决定能不能用"反常识"角度）
- **竞品占位**：竞品广告普遍在打哪个角度（决定"差异化空位"在哪）

#### 6.1.3 S2 角度空间扫描：6 大角度族

| 角度族 | 心理入口 | 适合 | 钩子感觉 |
|--------|----------|------|----------|
| **A 痛点放大** | 损失厌恶/焦虑 | 功能型/刚需 | "你是不是也……" |
| **B 反常识** | 认知失调/好奇 | 有误区的品类 | "其实你一直做错了" |
| **C 身份认同** | 归属/自我形象 | 颜值型/生活方式 | "懂的人都在用" |
| **D 省钱算账** | 经济理性 | 性价比/耐用 | "算笔账你就懂了" |
| **E 信任背书** | 权威/社会认同 | 高客单/破疑虑 | "我替你花2000块踩过坑" |
| **F 情绪共鸣** | 治愈/爽感/解压 | 情感型/礼赠 | "看完莫名很爽" |

每族至少扫 2 个候选角度，强制跨族覆盖。

#### 6.1.4 S3 钩子公式库（10 族 × 8 条 = 80 条）

| 钩子族 | 代表模板 | AI友好度 |
|--------|----------|----------|
| 痛点焦虑 H1 | "你是不是也{c Pain}？每次{scene}都{feeling}" | 🟢 0.95 |
| 好奇心 H2 | "99%的人不知道，{product}还能这么用" | 🟢 0.94 |
| 利益前置 H3 | "省{c Amount}块！学会这招再也不用{action}" | 🟢 0.95 |
| 反直觉 H4 | "别再{common}了！其实{truth}" | 🟢 0.94 |
| 模式打断 H5 | "停！先别滑走，给你看个好东西" | 🟡 0.88 |
| 身份认同 H6 | "真正的{c Role}，早就不{old_way}了" | 🟡 0.88 |
| 数字列表 H7 | "{c N}个{c Category}小技巧，第{c K}个最值钱" | 🟢 0.93 |
| 对比反差 H8 | "Before：{before} → After：{after}" | 🟢 0.92 |
| 情感共鸣 H9 | "那一刻我真的{c Emotion}了" | 🟡 0.88 |
| 互动提问 H10 | "你觉得{c Product}值{c Price}吗？" | 🟢 0.91 |

**钩子红线**：禁止"大家好今天推荐一款…"无效开场；前 3 秒必须有可见钩子。

#### 6.1.5 S5 WinScore 评分

```
WinScore = 角度新颖度(0-25)     ← 与竞品占位的距离，越空白越高
         + 痛点强度(0-20)       ← 痛点是否真实、是否高频
         + 钩子强度(0-20)       ← 前3秒能否拦住人
         + 平台契合(0-15)       ← 角度与平台调性匹配
         + 信任密度(0-10)       ← 是否易于用真实证据支撑
         + 卖点可视化(0-10)     ← 角度能否落到"看得见"
         − 同质化扣分 − 合规风险扣分
```

**诚实声明**：先验排序，不是真实预测。建议小预算实测 Top-3，回填 actual 再校准。

### 6.2 混剪模块（hybrid-cut）

> 实测真人 UGC 仍跑赢纯 AI（CTR 约 2.4% vs 1.9%）。最佳实践：**真实处用真素材建信任、难拍/贵/不可能处用 AI 补产能**。

#### 6.2.1 核心决策矩阵

| 场景 | 决策 | 原因 |
|------|------|------|
| 真人脸/口播、客户证言、真实效果对比 | **真素材 R** | 信任价值高，AI难以可信复刻 |
| 多场景快切、极限演示、不可能机位 | **AI 补拍** | 信任价值低，真拍太贵/不可能 |
| 真产品 + AI换背景、真口播音轨 + AI B-roll | **混合 H** | 兼顾信任与产能 |

**铁律**：证言、真实效果、真人脸 **永远用真**——这是信任的来源。

#### 6.2.2 成本铁律：AI 只做 0-3s 钩子

```
传统：整片30s全AI  → 贵（$1.5–6/条），还容易假
本法：AI只做 0–3s 钩子（最高杠杆的3秒），3s后全部【有序拼接真实素材】
     → AI成本砍到 1/10；真素材免费且更可信
```

#### 6.2.3 转化链路（8 要素）

```
痛点 → 旧法失败 → 救星登场 → 反差对比 → 产品接入 → 爽点展示 → 多场景 → CTA
```

### 6.3 矩阵内容模块（matrix-content）

> 2026 投放基线：认真投的账号每月要 15–30 条新素材。最大的坑是「一搬运/太像就限流」。

#### 6.3.1 精品母版法（Hero → Matrix）

```
第1步 雕母版（Hero）：先把 1 条做到满分精品——过完整 120 门质检。
第2步 提炼质量 DNA：把母版的【结构/节奏/卖点可视化/真实感参数/合规】固化为模板。
第3步 复用精品身体：矩阵 N 条 = 复用母版【已是精品的真素材身体】，只换【钩子】。
第4步 逐条过同一闸：每条变体独立过【同一120门标准】，不达标不发。
→ 不存在"矩阵专用的低标准"，闸门只有一个。
```

#### 6.3.2 4 大轮换轴

| 轮换轴 | 怎么换 | 作用 |
|--------|--------|------|
| 钩子轮换 | 同一卖点换 3-5 种钩子公式 | 测哪个开场最拦人 |
| 人设轮换 | 学生党/宝妈/上班族/专业测评… | 覆盖不同人群+账号差异 |

### 6.4 成本质量模块（cost-quality）

#### 6.4.1 成本现状（2026 实测，按秒计费）

| 模型 | 每秒价 | 10秒成本 | 备注 |
|------|--------|----------|------|
| Wan 2.6 | ~$0.05/s | ~$0.5 | 最低档 |
| Runway Gen-4 Turbo | ~$0.05/s | ~$0.5 | API |
| Veo 3.1 Lite | ~$0.05/s | ~$0.5 | |
| 可灵 Kling | ~$0.07/s | ~$0.7 | 性价比高 |
| Seedance 2 | ~$0.20/s | ~$2.0 | 国内强，带原生音→需静音 |
| Veo 3.1 Standard | ~$0.40/s | ~$7.5 | 最贵 |

**结论**：AI 视频的秒数 = 主要成本。30秒纯 AI 片 ≈ $1.5–6；一个 3 秒 AI 钩子 ≈ **$0.15–0.6**。差 10 倍。

#### 6.4.2 降本三招

1. **钱只花在钩子**：AI 只做 0-3s 钩子，3s 后有序拼接真实素材 → 省 90%
2. **能用 AI 图就不用 AI 视频**：静态镜用 AI 图 + Ken Burns 轻微推拉 → 再降一个数量级
3. **一次性投入，全矩阵摊薄**：声纹克隆一次/真素材库拍一次/精品母版雕一条 → N 条复用

#### 6.4.3 成本/质量旋钮

| 预算档 | AI 用量 | 画质 | 适合 |
|--------|---------|------|------|
| 极省 | 仅 AI 图钩子(Ken Burns) + 全真素材 | 720p | 跑量测款、个人卖家 |
| 标准 ★ | 3s AI 视频钩子 + 真素材身体 + @voice1 | 720-1080p | 主力矩阵 |
| 精品+ | 多个 AI 镜 + 真素材 + 口型同步数字人 | 1080p+ | 头部赢家、品牌片 |

### 6.5 语音一致性模块（voice-consistency）

#### 6.5.1 核心机制：声纹锁定 @voice1

```
本次会话声纹档案：
  @voice1 = 主口播声纹（全片唯一音色锚）
  来源：真人素材克隆 / 选定 TTS 音色 / 数字人音色
  规则：钩子、卖点、CTA——所有口播都用 @voice1，禁止混入 AI 原生音
```

**三条铁律**：
1. **单一声纹**：全片口播只允许一个音色（@voice1）
2. **AI 画面静音**：AI 生成的视频片段一律关掉原生音频
3. **统一配音轨**：整片口播文案写成一段连续脚本，用 @voice1 一次性合成 → 贴到画面上

#### 6.5.2 @voice1 决策树

```
有真人口播素材吗？
├─ 有 → 真人声就是基准
│   ├─ 全片都能用真人录？ → 全程真人配音（最自然，零克隆成本）
│   └─ 后面卖点/CTA没录到？ → 克隆出 @voice1，用克隆声补齐
│
└─ 没有真人口播 → 选一个 TTS/克隆音色当 @voice1，全片都用它
```

#### 6.5.3 声纹克隆方案

| 方案 | 成本 | 适合 |
|------|------|------|
| CosyVoice 3.0（阿里开源） | **免费**（自部署，~4GB 显存） | 量大、要压成本、要中文方言 |
| GPT-SoVITS（开源） | **免费** | 同上 |
| 火山引擎声音复刻 | 商用按量 | 抖音/TK 带货 |
| ElevenLabs | $5-22/月 | 出海/多语言 |

**降本要点**：克隆一次，全矩阵 N 条复用——不随条数线性增长。

#### 6.5.4 口型同步策略

| 情况 | 要不要口型同步 | 省钱做法 |
|------|---------------|----------|
| AI 钩子是产品/场景/**无人脸** | **不用** ★ | 直接叠 @voice1 旁白，最省 |
| AI 钩子有**说话的脸** | 要 | 用口型同步工具对到 AI 脸 |
| 真人素材本身在说话 | 不用 | 保留原声（=@voice1 基准） |

**成本铁律**：能不出现"AI 说话的脸"就别出现——**让 AI 钩子用无人脸的视觉打断镜**，旁白用 @voice1 叠上去。一举三得：省口型成本 + 躲恐怖谷 + 音色一致。

## 9. 进化协议

### 9.1 操作循环（每次开工/收工）

```
开工: params = loadParams()           // 读最新生效参数
       │                               // 必要时拉 HQ /params/latest 经门控
       ▼
跑流程: 用 params 里的角度权重/提示后缀/质检门 执行 S1~S8
       │
       ▼
收工: logSignals({                     // 匿名隐式信号
        scene, success, qcPass, retries, ms, model, angle
      })
       │
       ▼
定期: evolveRound(signals) →          // 跑一轮进化
       propose(signals) → validate() → applyLearningRate()
       │
       ├─ 通过验证 → commit → 更新 params → 下次更好
       └─ 拒绝 → 入负样本缓冲 → 保留当前 → 记录原因
```

### 9.2 手动执行进化

```bash
# 跑一轮本地进化
node bin/cli.js evolve

# 查看进化统计
cat .uvg-out/evolution/stats.json
```

### 9.3 编程调用

```javascript
const { EvolutionEngine, SkillParams, QualitySignals, Telemetry, VersionChecker } = require('./index');

// 1. 加载当前参数
const params = SkillParams.load('./.uvg-out/params.json');
const engine = new EvolutionEngine(params);

// 2. 跑完流程后记录信号
const signals = {
  success: true,       // 是否成功完成
  retries: 0,          // 重试次数
  regenerated: false,  // 是否重做
  qcScore: 110,        // 质检分数
  qcPass: true,        // 是否通过质检
  assembleOk: true,    // 拼接是否成功
  ms: 120000,          // 耗时(ms)
  model: 'kuaizi',     // 使用的模型
  scene: '痛点放大',    // 角度族
  angle: 'A'           // 具体角度
};

// 3. 跑一轮进化
const result = engine.evolveRound(signals);
if (result.success) {
  console.log('进化成功，新版本:', result.params.version);
  result.params.save('./.uvg-out/params.json');
} else {
  console.log('进化未通过:', result.reason);
}

// 4. 匿名遥测上报（opt-in，默认关闭）
const telemetry = new Telemetry({ enabled: process.env.UVG_TELEMETRY === 'on' });
telemetry.record('generate', signals);
telemetry.flush();  // 批量上报到 HQ

// 5. 检查总部更新
const versionChecker = new VersionChecker('https://hq.example.com/params/latest');
const latest = await versionChecker.fetchLatest();
if (latest.version > params.version) {
  const adopted = versionChecker.adopt(latest.params, recentEvents);
  if (adopted) console.log('已采纳 HQ 参数:', latest.version);
}

// 6. 一键回滚（进化跑偏时）
engine.rollback();
console.log('已回滚到:', engine.params.version);

// 7. 查看统计
console.log(engine.getStats());
// → { totalRounds: 45, evolves: 12, rejects: 30, rollbacks: 3, ... }
```

| `minSamples` | 10 | 验证门控最小样本数 |

| `weightCeiling` | 5.0 | 权重硬上限 |
| `weightFloor` | 0.1 | 权重硬下限 |

## 11. 边界与已知限制（诚实）

### 11.1 当前限制

| 限制 | 说明 |  workaround |
|------|------|-------------|
| 丽帧单条 ≤15s | 模型硬性限制 | >15s 由 ffmpeg 分段拼接 |
| 浏览器不能直连丽帧 | CORS 白名单无 `ApiKey` 头 | 必须经本地服务 (:8088) 中转 |
| 图生视频需公网 URL | 丽帧服务器抓不到本地文件 | 上传素材需先传到公网图床 |
| gpt-image-2pro 已废弃 | 网关返回 503 | 使用 `gpt-image-2` |
| 产品三视图非基于上传图 | 网关图生图不收 dataURI | 产品一致性靠丽帧首帧=上传图 |
| Kimi 新 key 401 | 旧 key 可用 | 优先 DeepSeek，Kimi 备选 |
| 续帧一致当前未启用 | 多段都用同一张上传图作首帧 | 有重复感，后续启用尾帧链式 |

### 11.2 待办优先级

1. **拆分超时**：文本/看图用短超时(30-40s)，只给画图留 120s
2. **真·产品三视图**：上传图传成 http 公网 URL 再作 reference_images
3. **接入完整 9 字段故事板**：把提示词总库的专业版故事板接入脚本和分镜图
4. **长视频续帧一致**：上一段尾帧 → 下一段首帧链式衔接
5. **错误透传到 UI**：seg 级错误透到 `/api/jobs` 的 segments
6. **可选模型**：接 sora-2、Kling Omni 作为视频备选
7. **TS 引擎合并**：src/ core/ adapters/ 早期 TS 引擎与 live-server.js 合并
